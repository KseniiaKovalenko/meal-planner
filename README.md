# 🍽️ Meal Planner — Планувальник їжі

> Персональний PWA-додаток для планування меню, зберігання рецептів та формування списку покупок. Працює офлайн, встановлюється на головний екран телефону без App Store.

---

## Можливості

- **Рецепти** — картки з фото, інгредієнтами, кроками приготування, КБЖВ та посиланням на джерело. Пошук і фільтрація по тегах
- **Теги замість категорій у рецептах** — рецепти більше не мають окремих категорій, для організації й фільтрації використовуються лише теги
- **Автододавання продуктів** — якщо інгредієнта ще немає в каталозі, він автоматично створюється в каталозі зі статусом `Новий`
- **Планувальник** — тижневий календар з 4 прийомами їжі (сніданок, обід, вечеря, перекус). Підтримка кількох тижнів і налаштування кількості осіб
- **Список покупок** — автогенерація з плану з урахуванням кількості порцій рецепта та кількості людей у планувальнику, групування по категоріях продуктів, позначення куплених
- **Комора** — список того що є вдома, з пошуком, автодоповненням з каталогу та полем `Придатний до`. Прострочені позиції підсвічуються червоним
- **Каталог продуктів** — база продуктів з КБЖВ, категорією, тегами та посиланням на магазин. Для невагових одиниць підтримується логіка на кшталт `1 шт = 62 г`, щоб коректно рахувати КБЖВ у рецептах
- **Резервна копія** — експорт/імпорт всіх даних у JSON-файл для переносу між пристроями
- **Адаптивний desktop UI** — окремі покращення для ноутбука й desktop у коморі та планувальнику

## Технології

| Що | Навіщо |
|---|---|
| Vanilla JS | Без фреймворків — один набір файлів, жодних залежностей |
| IndexedDB | Локальне сховище на пристрої |
| Service Worker | Офлайн-режим та кешування |
| Web App Manifest | Встановлення на головний екран |
| GitHub Pages | Безкоштовний хостинг |

## Структура проєкту

```
meal-planner/
├── index.html        # HTML-оболонка
├── manifest.json     # PWA конфіг
├── sw.js             # Service Worker (офлайн-кеш)
├── .nojekyll         # Вимикає Jekyll на GitHub Pages
├── css/
│   └── app.css       # Всі стилі
└── js/
    ├── constants.js  # Константи: одиниці, теги, продукти за замовч.
    ├── icons.js      # SVG іконки
    ├── db.js         # IndexedDB: відкриття БД, CRUD-операції
    ├── utils.js      # Спільні хелпери: toast, modal, форматування
    ├── recipes.js    # Вкладка «Рецепти»
    ├── planner.js    # Вкладка «Планувальник»
    ├── shopping.js   # Вкладка «Список покупок»
    ├── pantry.js     # Вкладка «Комора» + Каталог продуктів
    ├── backup.js     # Експорт / Імпорт JSON
    └── app.js        # Навігація, Service Worker, ініціалізація
```

## Встановлення та запуск

### Локально

```bash
git clone https://github.com/kseniiakovalenko/meal-planner.git
cd meal-planner

# Запустити локальний сервер (потрібен для Service Worker)
npx serve .
# або
python3 -m http.server 8080
```

Відкрити у браузері: `http://localhost:8080`

> ⚠️ Відкриття через `file://` не підтримує Service Worker. Потрібен локальний HTTP-сервер або розгортання на хостингу.

### GitHub Pages

1. Завантажте всі файли у репозиторій зі збереженням структури папок
2. **Settings → Pages → Branch: main → / (root) → Save**
3. Через 1–2 хвилини сайт буде доступний за адресою:
   `https://kseniiakovalenko.github.io/meal-planner`

## Встановлення як PWA на телефон

**iPhone (Safari):**
Кнопка «Поділитися» → «На Початковий екран» → «Додати»

**Android (Chrome):**
Меню ⋮ → «Додати на головний екран»

Після встановлення додаток відкривається без адресного рядка і працює повністю офлайн.

## Дані та приватність

- Всі дані зберігаються **локально на пристрої** у IndexedDB
- Жодні дані не передаються на сервер
- Для переносу між пристроями використовуйте **Експорт/Імпорт** (кнопка збереження у шапці)
- При оновленні фронтенду PWA може використовувати кешований код Service Worker. Якщо зміни не видно одразу, перезавантажте застосунок або оновіть сторінку

## Як це працює

- **Порції в рецепті**: поле `Порцій` задає базову кількість порцій рецепта. КБЖВ у рецепті зберігаються і показуються `на 1 порцію`
- **Люди в планувальнику**: кількість людей у тижні використовується для масштабування інгредієнтів при генерації списку покупок
- **Масштабування покупок**: формула така: `кількість людей / порції рецепта`. Наприклад, якщо рецепт розрахований на 4 порції, а в планувальнику вибрано 2 людини, у список покупок піде половина інгредієнтів
- **КБЖВ для штук**: якщо продукт використовується в `шт`, `скибка`, `упаковка` тощо, у каталозі можна вказати, скільки грамів або мл містить 1 одиниця. Це потрібно для правильного автопідрахунку КБЖВ
- **Статус `Новий`**: автоматично ставиться продуктам, які були створені з рецепта або комори, і зникає після першого ручного редагування продукту

## Резервна копія

```
Шапка додатку → іконка збереження → Експортувати дані
```

Файл `menu-backup-YYYY-MM-DD.json` містить усі рецепти, продукти, план та комору. Для відновлення — «Імпортувати дані» на будь-якому пристрої.
