# 🍽️ Meal Planner — Планувальник їжі

> Персональний PWA-додаток для планування меню, зберігання рецептів та формування списку покупок. Працює офлайн, встановлюється на головний екран телефону без App Store.

---

## Можливості

- **Рецепти** — картки з фото, інгредієнтами, кроками приготування, КБЖВ та посиланням на джерело. Пошук і фільтрація по тегах
- **Планувальник** — тижневий календар з 4 прийомами їжі (сніданок, обід, вечеря, перекус). Підтримка кількох тижнів і налаштування кількості осіб
- **Список покупок** — автогенерація з плану з урахуванням кількості порцій, групування по категоріях продуктів, позначення куплених
- **Комора** — список того що є вдома. Інгредієнти з комори відмічаються в списку покупок автоматично
- **Каталог продуктів** — база продуктів з КБЖВ/100г, категорією та посиланням на магазин. Використовується для авторозрахунку калорій рецепту
- **Резервна копія** — експорт/імпорт всіх даних у JSON-файл для переносу між пристроями

---

## Технології

| Що | Навіщо |
|---|---|
| Vanilla JS | Без фреймворків — один набір файлів, жодних залежностей |
| IndexedDB | Локальне сховище на пристрої |
| Service Worker | Офлайн-режим та кешування |
| Web App Manifest | Встановлення на головний екран |
| GitHub Pages | Безкоштовний хостинг |

## Структура проєкту

```
meal-planner/
├── index.html        # HTML-оболонка
├── manifest.json     # PWA конфіг
├── sw.js             # Service Worker (офлайн-кеш)
├── .nojekyll         # Вимикає Jekyll на GitHub Pages
├── css/
│   └── app.css       # Всі стилі
└── js/
    ├── constants.js  # Константи: одиниці, теги, категорії, продукти за замовч.
    ├── icons.js      # SVG іконки
    ├── db.js         # IndexedDB: відкриття БД, CRUD-операції
    ├── utils.js      # Спільні хелпери: toast, modal, форматування
    ├── recipes.js    # Вкладка «Рецепти»
    ├── planner.js    # Вкладка «Планувальник»
    ├── shopping.js   # Вкладка «Список покупок»
    ├── pantry.js     # Вкладка «Комора» + Каталог продуктів
    ├── backup.js     # Експорт / Імпорт JSON
    └── app.js        # Навігація, Service Worker, ініціалізація
```

## Встановлення та запуск

### Локально

```bash
git clone https://github.com/YOUR_USERNAME/meal-planner.git
cd meal-planner

# Запустити локальний сервер (потрібен для Service Worker)
npx serve .
# або
python3 -m http.server 8080
```

Відкрити у браузері: `http://localhost:8080`

> ⚠️ Відкриття через `file://` не підтримує Service Worker. Потрібен локальний HTTP-сервер або розгортання на хостингу.

### GitHub Pages

1. Завантажте всі файли у репозиторій зі збереженням структури папок
2. **Settings → Pages → Branch: main → / (root) → Save**
3. Через 1–2 хвилини сайт буде доступний за адресою:
   `https://YOUR_USERNAME.github.io/meal-planner`

## Встановлення як PWA на телефон

**iPhone (Safari):**
Кнопка «Поділитися» → «На екран "Домів"» → «Додати»

**Android (Chrome):**
Меню ⋮ → «Додати на головний екран»

Після встановлення додаток відкривається без адресного рядка і працює повністю офлайн.

## Дані та приватність

- Всі дані зберігаються **локально на пристрої** у IndexedDB
- Жодні дані не передаються на сервер
- Для переносу між пристроями використовуйте **Експорт/Імпорт** (кнопка збереження у шапці)

---

## Резервна копія

```
Шапка додатку → іконка збереження → Експортувати дані
```

Файл `menu-backup-YYYY-MM-DD.json` містить усі рецепти, продукти, план та комору. Для відновлення — «Імпортувати дані» на будь-якому пристрої.
